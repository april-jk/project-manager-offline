# Project Hub - 用户使用指南

## 介绍

**Project Hub** 是一个完全离线的项目管理工具，帮助您在一个统一的平台上管理多个项目、网站资源和敏感信息。所有数据存储在您的浏览器本地，确保隐私和安全。

### 主要特性

- **项目管理**: 创建和组织多个项目
- **网站黄页**: 以卡片形式展示和管理网站链接
- **备忘录**: 记录 API Key、密码等重要信息
- **加密存储**: 使用密码加密敏感数据
- **数据导出/导入**: 备份和恢复数据
- **完全离线**: 无需网络连接，数据永不上传

## 快速开始

### 1. 创建第一个项目

1. 点击首页右上角的"新建项目"按钮
2. 填写项目名称（必填）和描述（可选）
3. 选择一个颜色标签来区分项目
4. 点击"保存"按钮

项目创建后，您会看到它出现在首页的项目列表中。

### 2. 添加网站到项目

1. 点击项目卡片进入项目详情页
2. 点击"📌 网站黄页"标签页
3. 点击"添加网站"按钮
4. 填写以下信息：
   - **网站名称**: 例如 "GitHub"
   - **网站 URL**: 例如 "https://github.com"
   - **描述**: 可选，用于备注网站用途
   - **标签**: 可选，用逗号分隔多个标签

5. 点击"保存"按钮

网站添加后，您可以：
- **打开网站**: 悬停在卡片上，点击"打开"按钮在新标签页中打开
- **编辑网站**: 点击编辑按钮修改信息
- **删除网站**: 点击删除按钮移除网站

### 3. 记录备忘录

1. 在项目详情页点击"📝 备忘录"标签页
2. 点击"添加备忘录"按钮
3. 填写以下信息：
   - **标题**: 备忘录的标题（必填）
   - **内容**: 备忘录的具体内容
   - **分类**: 可选，例如 "API Key"、"密码"、"笔记"
   - **🔒 加密存储**: 如果已设置密码，可勾选此项加密存储

4. 点击"保存"按钮

## 安全功能

### 设置密码

为了保护敏感信息，您可以设置一个主密码来加密备忘录。

**设置步骤**:

1. 点击首页右上角的菜单按钮（三个点）
2. 选择"设置密码"
3. 输入至少 8 个字符的密码
4. 点击"设置密码"按钮

**密码要求**:
- 至少 8 个字符
- 建议包含大小写字母、数字和特殊字符
- 密码不会被存储，每次使用时由您输入

### 加密备忘录

1. 创建或编辑备忘录时，勾选"🔒 加密存储"选项
2. 备忘录内容将使用您的主密码进行加密
3. 加密的备忘录会显示一个锁形图标

### 解锁加密内容

1. 如果您关闭浏览器或刷新页面，加密内容会被锁定
2. 点击首页右上角的"解锁"按钮
3. 输入您的主密码
4. 点击"解锁"按钮

## 数据管理

### 导出数据

定期备份您的数据以防止意外丢失。

**导出步骤**:

1. 点击首页右上角的菜单按钮
2. 选择"导出数据"
3. 一个 JSON 文件会被下载到您的计算机
4. 妥善保管这个文件

### 导入数据

从备份文件恢复数据。

**导入步骤**:

1. 点击首页右上角的菜单按钮
2. 选择"导入数据"
3. 选择之前导出的 JSON 文件
4. 数据会被导入并覆盖现有数据
5. 页面会自动刷新

### 清空所有数据

**警告**: 此操作无法撤销！

1. 点击首页右上角的菜单按钮
2. 选择"清空所有数据"
3. 确认删除
4. 所有数据会被永久删除

## 常见问题

### Q: 我的数据会被上传到服务器吗？
**A**: 不会。Project Hub 是完全离线的应用，所有数据都存储在您的浏览器本地存储中，永不上传。

### Q: 我忘记了密码怎么办？
**A**: 遗憾的是，密码无法恢复。密码只存储在您的脑海中，如果忘记，加密的备忘录将无法解锁。建议您妥善保管密码。

### Q: 我可以在不同的浏览器中使用我的数据吗？
**A**: 可以，但需要手动导出和导入。每个浏览器都有自己的本地存储，所以您需要：
1. 在原浏览器中导出数据
2. 在新浏览器中导入数据

### Q: 清空浏览器缓存会删除我的数据吗？
**A**: 如果您清空"Cookie 和网站数据"，会删除 LocalStorage 中的所有数据。建议在清空缓存前先导出数据。

### Q: 网站图标为什么不显示？
**A**: 某些网站可能不支持 Favicon API，或者您的网络连接可能受限。在这种情况下，会显示默认的地球图标。

### Q: 我可以在手机上使用吗？
**A**: 可以。Project Hub 是响应式设计，支持手机、平板和桌面浏览器。

## 最佳实践

1. **定期备份**: 每周导出一次数据，保存在安全的地方
2. **强密码**: 设置复杂的密码来保护敏感信息
3. **组织项目**: 为不同的工作创建不同的项目，便于管理
4. **使用标签**: 为网站添加标签，方便搜索和分类
5. **及时更新**: 定期更新过期的网站链接和信息

## 隐私和安全

- **本地存储**: 所有数据存储在浏览器的 LocalStorage 中
- **加密算法**: 使用 AES-256-GCM 加密敏感数据
- **密钥派生**: 使用 PBKDF2 从密码派生加密密钥
- **无网络传输**: 应用不会向任何服务器发送数据

## 技术细节

### 存储结构

数据以 JSON 格式存储在浏览器的 LocalStorage 中：

- `pm-projects`: 项目列表
- `pm-websites`: 网站列表
- `pm-memos`: 备忘录列表
- `pm-app-settings`: 应用设置

### 加密过程

1. 用户输入主密码
2. 使用 PBKDF2 从密码派生 256 位密钥
3. 使用 AES-256-GCM 加密备忘录内容
4. 每次加密生成新的随机 IV 和 Salt
